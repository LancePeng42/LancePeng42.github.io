Git 基础

配置 git

添加配置

git config [--local | --global | --system] user.name 'Your name'

git config [--local | --global | --system] user.email 'Your email'

查看配置

git config --list [--local | --global | --system]

清除配置

git config unset [--local | --global | --system] user.name

区别

  • local：区域为本仓库,配置信息记录在.git/config 中
  • global: 当前用户的所有仓库,配置信息记录个人 home 目录下的.gitconfig 中
  • system: 本系统的所有用户,配置信息记录在.git/configgit 安装目录的中

优先级 local > global > system

创建 Git 仓库

两种⽅式：

  1. ⽤ Git 之前已经有项⽬代码
  1. ⽤ Git 之前还没有项⽬代码

往仓库⾥添加⽂件

git add . | -A:将当前目录或其子目录下相应文件的修改、删除，添加到暂存区。

git add -u:将工作目录被修改和被删除的文件添加到暂存区(不包含没有纳入 Git 管理的新增文件)

git commit -m "Some note about this commit"

通过 git log 查看版本演变历史

git log --all: 查看所有分支的历史

git log --all --graph: 查看图形化的 log 地址

git log --oneline： 查看单行的简洁历史。

git log --oneline -n4: 查看最近的四条简洁历史。

git log --oneline --all -n4 --graph：查看所有分支最近 4 条单行的图形化历史。

gitk:通过图形界面工具来查看版本历史

探秘.git 目录

.git 文件目录：

  • COMMIT_EDITMSG

  • config 当前 git 的配置文件

  • description （仓库的描述信息文件）

  • HEAD （指向当前所在的分支），例如当前在 develop 分支，实际指向地址是 refs/heads/develop

  • ORIG_HEAD/FETCH_HEAD

  • index

  • heads （存放当前项目的所有分支）

  • tags (存放的当前项目的所有标签，又叫做里程碑)

  • hooks [文件夹]

  • info [文件夹]

  • logs [文件夹]

  • objects [文件夹] （存放所有的 git 对象，对象哈希值前 2 位作为文件夹名称，后 38 位作为对象文件名, 可通过 git cat-file -p 命令，拼接文件夹名称+文件名查看）

  • refs [文件夹]

git cat -file -t: 查看 git 对象的类型

git cat -file -p: 查看 git 对象的内容

git cat -file -s: 查看 git 对象的大小

commit、tree 和 blob 三个对象之间的关系

git 有 3 种对象：commit、tree、blob。

每次的提交，都是一个 commit，一个 commit 中包含了一棵 tree、author、message 等信息，每个 tree 中可递归包含多棵 tree 和 blob，而文件的最终形式是 blob。

git 对于文件内容相同的 blob，采用享元模式，使用同一个 blob。不同的 commit 的区别是 commit、tree 和有差异的 blob，多数未变更的文件对应的 blob 都是相同的，这么设计对于版本管理系统来说可以省很多存储空间。其次，Git 还有增量存储的机制。
