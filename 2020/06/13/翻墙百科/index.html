翻墙百科

Posted on 2020-06-13 | In 生存技能

概述

墙是什么

墙是防火长城，GFW(Great Fire Wall)。主要作用在于分析和过滤境外咨询和互相访问，是审查制度下的产物，总体利大于弊。

GFW 工作原理

GFW 封锁方式

  • DNS 污染，解析错误地址或者不解析
  • 过滤关键字
  • 端口阻断(Port:443，22)
  • IP 地址批量封锁

翻墙工作原理

初代 VPN

只要 GFW 不探明所发送数据包的真实意图，那么就可以通过连接一台符合 GFW 认为合理的服务器作为中转，来替代用户去做想做的事情，然后返回结果，全程加密。这是很多初期 http 代理 VPN、SSH 这些，突破封锁的实现方式。

  • 数据流量不再是明文，而是加密数据，GFW 看不到关键字
  • DNS 解析时，发现用户访问的对象是允许的服务器而不是 google

但 GFW 发现，越来越多的流量都是这种类似的模式：先发送一个数据包请求和一个服务器建立加密连接，然后紧跟着一个代理请求，这就是传说中的特征。
VPN 和 http 代理曾经一度非常流行，后来由于太多用户使用这种方式来翻墙，所以 GFW 根据这些特别有特征明显的这种流量做了两件事：

  • VPN 通讯用的端口被封锁，政企用户需要申报审核
  • 提供 VPN 服务的服务器 IP 地址批量被封

Shadowsocks

Shadowsocks 引出了一个把代理服务器拆分成本地和远程两个的这种概念，实现经过 GFW 的流量全部加密，从而消除明显的流量特征这个概念，后来衍生出来的 SSR、v2ray 这些都是基于类似的原理：

GFW 之所以不封锁所有的国际加密流量，是因为相比翻墙的流量，超过 99%的通信流量都是正常的需求。所以目前为止，Shadowsocks 这种实现原理相对安全。

多种软硬件翻墙方式

软件翻墙

VPN(Virtual Private Network)

vpn 只是翻墙的一种方式，虚拟私有网络，最早应用于一些大公司中，因私密通信的需求而产生，是一种端对端的虚拟私有专线，可避免通讯流量被中间人劫持。故最早并不是应用于翻墙，被流行用于翻墙后，因其流量特征明显，早就可以被 GFW 完美探测并封锁，故私人用户用 VPN 翻墙并不安全，只有申请报备的企业用户才可正常使用。

Shadowsocks 及衍生品(SSR V2RAY Trojan)

该类软件利用本地服务器加密流量，然后将加密流量发送至远程服务器解密，再进行转发，进而突破 GFW 的封锁。GFW 封锁网络，不会去解密流量，而是通过各种方式来判断该流量的意图，所以无论采用那种加密协议都不是完全安全的。

这些协议运行于应用层的应用软件，常见的应用软件有 Shadowsocks、ShadowsocksR、Shadowrocket、V2Ray、Clash、Surge。越新的协议和加密算法意味着相对安全，越复杂的加解密算法需要较高的算力，不适合算力差的路由器，相对的，旧的协议也有其适用的场景。

该类软件翻墙方式局限：

大多数软件没有一定的底层权限，只能代理应用层的网络流量。例如：即使打开全局代理模式，某些软件的通讯流量依然无法代理。常见于 UWP 应用，其运行于系统层，具有较高的权限，甚至不遵循操作系统的代理规则，不将通讯流量交给翻墙软件接管。有些游戏应用也是如此，其通讯协议是 TCP UDP，翻墙软件无法接管其通讯流量，自然也无法加速。

少数软件功能强大，可模拟虚拟网卡，强制接管所有流量，能够实现真的全局代理。

硬件翻墙（路由器翻墙）

硬路由

该方式可无视各种应用层网络层硬件层的通讯协议，实现真的全局代理。路由器是本地网络接入互联网的最后一个环节，并且独立于电脑，所以可以无视流量代理规则，强制接收所有流量，来实现真全局代理。

原理可简单理解为将翻墙软件部署在了路由器上，连接路由器的设备都可以通过路由器翻墙。

优点：可应用于无法安装翻墙软件的设备上

缺点：路由器算力弱，基本只能满足最基础的路由功能，大量的加解密运算会使网络通讯产生卡顿

软路由

软路由是改装过的路由器，算力更强，克服了硬路由算力不足的短板。

缺点：价格贵；配置繁琐；稳定性一般；维护繁琐

网关模式翻墙

通过一台计算机上跑的软件来实现路由器功能，代表性产品：Windows 下的 Clash，Mac 下的 Surge。在本地局域网里部署一台运行了 Surge 或 Clash 的电脑当作网关，可接管局域网里所有网络的设备的所有流量。

优点：方便，效率高；性能良好；便于集中管理
