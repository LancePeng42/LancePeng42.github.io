利用ss代理配合Proxifier解决OneNote同步问题

Posted on 2019-09-03 | In 计算机网络

同步问题

OneNote，功能强大，是 Office 家族的一员，而且还免费。唯一遗憾的是，存在同步，伟大的 GFW！

个人版 OneNote 使用的 OneDrive 来同步笔记，然而受 DNS 污染等因素的影响，国内同步一直是个老大难。如果不差钱的话，可购买 OneDrive for Bussiness，商业版在国内有服务器，据说速度飞快。

准备工作

  1. 代理服务，类似 ssh,纸飞机类的代理请自备
  2. Proxifier 官方下载地址

Proxifier 为收费软件，这里提供两个注册码, 软件分为 Standard Edition 和 Portable Edition 版本，注册码不通用，注册用户名任意。

配置 Proxifier

  1. 配置代理服务器。
  • 首先打开“Proxifier”软件，然后配置文件 → 代理服务器（或者点击下图中的快捷图标）
  • 地址：127.0.0.1（这个都一样）
  • 端口：1080（你所使用的代理端口，一般是 1080）
  • 协议：SOCKS 5
  • 其他不填，然后点“确定”即可

  1. 设置规则
  • 配置文件→代理规则，进行添加规则。
  • 默认是有个名称为 Default 的规则，将其动作设为 Direct ,使电脑全部软件直接连接网络,而不通过 SS 代理联网
  • 然后点添加规则，添加一项局部代理规则，让想走代理线路的软件走代理。
    名称：任意
  • 应用程序：需要走代理的软件，这里需要代理的是 OneNote.exe
  • 动作：Proxy SOCKS5 127.0.0.1

配置好的规则列表:

  1. 设置远程 DNS

最后开启远程 DNS 解析，防止本地 DNS 污染
Profile→Name Resolution 或者点击软件左上角的 DNS，取消勾选第一个选项，选择第二个选项

  1. 运行 SS 代理

以后有什么软件需要使用代理就在 Proxifier 规则里添加，不需要走代理则去除该应用即可，也可以添加多项规则进行勾选来切换，这样就不用来回添加和删除某些应用。

  • 利用代理配合 Proxifier 给 Onenote 同步加速

服务器环境搭建

服务器软件

HTTP Server 运行于服务器之上，主要工作有：

  • 监听端口
  • 接收 socket 连接
  • 解析 HTTP 请求
  • 使用通用或专用协议对请求进行分发
  • 接收分发的请求产生的运行结果
  • 将结果格式化成 HTTP Response 并写到 socket 里面
  • 关闭连接或者 Keep-Alive

采用动静态资源分离的方式，选用 Nginx 处理静态资源请求，Tomcat 处理动态资源请求。

Ngnix 安装

WEB 服务器，处理静态资源请求的同时，将动态请求反向代理至 Tomcat。

Ubuntu:

sudo apt-get install nginx

CentOS:

sudo yum install nginx

检查存储字符集

输入sudo mysql -u root -p并输入密码后，进入 mysql 命令行界面

提示需要密码时，输入安装时设定的密码;若未设定，密码则为空;

剩下的情况需以维护模式进入 mysql，输入mysql -u debian-sys-maint -p,并以/etc/mysql/debian.cnf文件中的密码登录，以设定或修改 root 的密码：

成功进入 mysql 命令行界面后，分别输入以下命令检查字符集编码:

show variables like '%character%';

show variables like 'collation%';

如编码存在瑞典 latin1，需修改为 utf-8:
向/etc/mysql/my.cnf(不一定是此文件，不同版本不同)中 mysqld 部分添加default-character-set = utf8并重启 mysql。

配置远程登录

  1. 登录 mysql

    sudo mysql -u root -p

  2. 创建新用户

    CREATE USER 'username' IDENTIFIED BY 'password';

  3. 授予远程登录权限

    GRANT ALL ON *.* TO 'username'@'%' IDENTIFIED BY 'password'

其中*.*为授权的数据库，格式为 dbname.tablename，此处为全部授权;
username 为用户名， %意味着允许用户发起远程访问的 IP 为任意 IP，也可以指定 password 为用户密码。

  1. 修改 mysql 配置文件

将/etc/mysql/my.cnf中bind_address=127.0.0.1值改为0.0.0.0。
