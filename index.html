CheckStyle 与 CheckStyle-IDEA 联系

CheckStyle

Checkstyle 是一种开发工具，可帮助程序员编写符合编码标准的Java代码。Checkstyle 可以检查源代码的许多方面。它可以发现类设计问题，方法设计问题。它还能够检查代码布局和格式问题。

可将CheckStyle通过 Ant 或 CLI 的方式集成到项目的构建过程或开发环境中，进行使用。

CheckStyle-IDEA

为方便使用，众多第三方开发者便基于CheckStyle，针对主流IDE，开发出了相应的插件。CheckStyle-IDEA 便是IntelliJ IDEA平台的插件工具，提供代码检测的实时反馈。

可在IntelliJ IDEA 插件市场中安装使用。

IntelliJ IDEA 与 IntelliJ Platform 简介

IntelliJ IDEA 简称 IDEA，是 Jetbrains 公司旗下的一款 JAVA 开发工具，支持 Java、Scala、Groovy 等语言的开发，同时具备支持目前主流的技术和框架，擅长于企业应用、移动应用和 Web 应用的开发，提供了丰富的功能，智能代码助手、代码自动提示、重构、J2EE 支持、各类版本工具(git、svn 等)、JUnit、CVS 整合、代码分析、 创新的 GUI 设计等。

IntelliJ Platform 是一个构建 IDE 的开源平台，基于它构建的 IDE 有 IntelliJ IDEA、WebStorm、DataGrip、以及 Android Studio 等等。IDEA 插件也是基于 IntelliJ Platform 开发的。

开发环境搭建

  1. 下载并安装 IntelliJ IDEA

下载并安装 IntelliJ IDEA Ultimate 或 IntelliJ IDEA Community Edition.

  • 社区版（Community）：完全免费，代码开源，但是缺少一些旗舰版中的高级特性

  • 旗舰版（Ultimate）：30 天免费，支持全部功能，代码不开源

相传开发 IDEA 的插件推荐使用社区版，因为社区版是开源的，在开发插件的时候，可以调试源代码。

  1. 确保启用“Plugin DevKit”插件

Plugin DevKit 是 IntelliJ 的一个插件，它使用 IntelliJ IDEA 自己的构建系统来为开发 IDEA 插件提供支持。开发 IDEA 插件之前需要安装并启用 Plugin DevKit 。

打开 IDEA，导航到 Settings | Plugins，若插件列表中没有 Plugin DevKit，点击 Install JetBrains plugin，搜索并安装。

创建一个 Action

Action 是 IDEA 中对事件响应的处理器，它的 actionPerformed 就像是 JS 中的 onClick 方法。可以看出来，插件的开发本质上跟 web、Android 的开发没有什么不同，因为都是事件驱动的编程。

可以直接使用 IDEA 提供的 Action 生成器

插件 New Action 说明如下：

  • id：作为标签的唯一标识。一般以<项目名>.<类名>方式。
  • class：即我们自定义的 AnAction 类
  • text：显示的文字，如我们自定义的插件放在菜单列表中，这个文字就是对应的菜单项
  • description：对这个 AnAction 的描述
  • Groups：表示插件入口出现的位置，比如可以让插件出现在 Tools 菜单上，则可以选择 ToolsMenu,如果想让插件出现在编辑框 Generate 菜单则可以选择 GenerateGroup
  • Actions: 已有的 Action，即已有的插件功能，选定这里的 Action 配合 Anchor 选项，可以指定我们新建的 Action 出现在已有的 Action 之前或是之后。
  • Anchor：用来指定动作选项在 Groups 中的位置，Frist 就是最上面、Last 是最下面，也可以设在某个选项的上/下方
  • Keyboard Shortcuts：调用插件 Action 的快捷键，可以不填，填了要注意热键冲突

填写完必要信息后，可以看到 resources/META-INF/plugin.xml 文件多了一个 Actions 节点，新建的 Action 添加到了 Actions 节点中。

新建 Action 继承 AnAction 类，插件实现的操作实现 actionPerformed 方法即可。

Git-Learning

Posted on 2020-08-05 | In 开发工具

Git 基础

配置 git

添加配置

git config [--local | --global | --system] user.name 'Your name'

git config [--local | --global | --system] user.email 'Your email'

查看配置

git config --list [--local | --global | --system]

清除配置

git config unset [--local | --global | --system] user.name

区别

  • local：区域为本仓库,配置信息记录在.git/config 中
  • global: 当前用户的所有仓库,配置信息记录个人 home 目录下的.gitconfig 中
  • system: 本系统的所有用户,配置信息记录在.git/configgit 安装目录的中

优先级 local > global > system

创建 Git 仓库

两种⽅式：

  1. ⽤ Git 之前已经有项⽬代码
  1. ⽤ Git 之前还没有项⽬代码

往仓库⾥添加⽂件

git add . | -A:将当前目录或其子目录下相应文件的修改、删除，添加到暂存区。

git add -u:将工作目录被修改和被删除的文件添加到暂存区(不包含没有纳入 Git 管理的新增文件)

git commit -m "Some note about this commit"

通过 git log 查看版本演变历史

git log --all: 查看所有分支的历史

git log --all --graph: 查看图形化的 log 地址

git log --oneline： 查看单行的简洁历史。

git log --oneline -n4: 查看最近的四条简洁历史。

git log --oneline --all -n4 --graph：查看所有分支最近 4 条单行的图形化历史。

gitk:通过图形界面工具来查看版本历史

探秘.git 目录

.git 文件目录：

  • COMMIT_EDITMSG

  • config 当前 git 的配置文件

  • description （仓库的描述信息文件）

  • HEAD （指向当前所在的分支），例如当前在 develop 分支，实际指向地址是 refs/heads/develop

  • ORIG_HEAD/FETCH_HEAD

  • index

  • heads （存放当前项目的所有分支）

  • tags (存放的当前项目的所有标签，又叫做里程碑)

  • hooks [文件夹]

  • info [文件夹]

  • logs [文件夹]

  • objects [文件夹] （存放所有的 git 对象，对象哈希值前 2 位作为文件夹名称，后 38 位作为对象文件名, 可通过 git cat-file -p 命令，拼接文件夹名称+文件名查看）

  • refs [文件夹]

git cat -file -t: 查看 git 对象的类型

git cat -file -p: 查看 git 对象的内容

git cat -file -s: 查看 git 对象的大小

commit、tree 和 blob 三个对象之间的关系

git 有 3 种对象：commit、tree、blob。

每次的提交，都是一个 commit，一个 commit 中包含了一棵 tree、author、message 等信息，每个 tree 中可递归包含多棵 tree 和 blob，而文件的最终形式是 blob。

git 对于文件内容相同的 blob，采用享元模式，使用同一个 blob。不同的 commit 的区别是 commit、tree 和有差异的 blob，多数未变更的文件对应的 blob 都是相同的，这么设计对于版本管理系统来说可以省很多存储空间。其次，Git 还有增量存储的机制。

翻墙百科

Posted on 2020-06-13 | In 生存技能

概述

墙是什么

墙是防火长城，GFW(Great Fire Wall)。主要作用在于分析和过滤境外咨询和互相访问，是审查制度下的产物，总体利大于弊。

GFW 工作原理

GFW 封锁方式

  • DNS 污染，解析错误地址或者不解析
  • 过滤关键字
  • 端口阻断(Port:443，22)
  • IP 地址批量封锁

翻墙工作原理

初代 VPN

只要 GFW 不探明所发送数据包的真实意图，那么就可以通过连接一台符合 GFW 认为合理的服务器作为中转，来替代用户去做想做的事情，然后返回结果，全程加密。这是很多初期 http 代理 VPN、SSH 这些，突破封锁的实现方式。

  • 数据流量不再是明文，而是加密数据，GFW 看不到关键字
  • DNS 解析时，发现用户访问的对象是允许的服务器而不是 google

但 GFW 发现，越来越多的流量都是这种类似的模式：先发送一个数据包请求和一个服务器建立加密连接，然后紧跟着一个代理请求，这就是传说中的特征。
VPN 和 http 代理曾经一度非常流行，后来由于太多用户使用这种方式来翻墙，所以 GFW 根据这些特别有特征明显的这种流量做了两件事：

  • VPN 通讯用的端口被封锁，政企用户需要申报审核
  • 提供 VPN 服务的服务器 IP 地址批量被封

Shadowsocks

Shadowsocks 引出了一个把代理服务器拆分成本地和远程两个的这种概念，实现经过 GFW 的流量全部加密，从而消除明显的流量特征这个概念，后来衍生出来的 SSR、v2ray 这些都是基于类似的原理：

GFW 之所以不封锁所有的国际加密流量，是因为相比翻墙的流量，超过 99%的通信流量都是正常的需求。所以目前为止，Shadowsocks 这种实现原理相对安全。

多种软硬件翻墙方式

软件翻墙

VPN(Virtual Private Network)

vpn 只是翻墙的一种方式，虚拟私有网络，最早应用于一些大公司中，因私密通信的需求而产生，是一种端对端的虚拟私有专线，可避免通讯流量被中间人劫持。故最早并不是应用于翻墙，被流行用于翻墙后，因其流量特征明显，早就可以被 GFW 完美探测并封锁，故私人用户用 VPN 翻墙并不安全，只有申请报备的企业用户才可正常使用。

Shadowsocks 及衍生品(SSR V2RAY Trojan)

该类软件利用本地服务器加密流量，然后将加密流量发送至远程服务器解密，再进行转发，进而突破 GFW 的封锁。GFW 封锁网络，不会去解密流量，而是通过各种方式来判断该流量的意图，所以无论采用那种加密协议都不是完全安全的。

这些协议运行于应用层的应用软件，常见的应用软件有 Shadowsocks、ShadowsocksR、Shadowrocket、V2Ray、Clash、Surge。越新的协议和加密算法意味着相对安全，越复杂的加解密算法需要较高的算力，不适合算力差的路由器，相对的，旧的协议也有其适用的场景。

该类软件翻墙方式局限：

大多数软件没有一定的底层权限，只能代理应用层的网络流量。例如：即使打开全局代理模式，某些软件的通讯流量依然无法代理。常见于 UWP 应用，其运行于系统层，具有较高的权限，甚至不遵循操作系统的代理规则，不将通讯流量交给翻墙软件接管。有些游戏应用也是如此，其通讯协议是 TCP UDP，翻墙软件无法接管其通讯流量，自然也无法加速。

少数软件功能强大，可模拟虚拟网卡，强制接管所有流量，能够实现真的全局代理。

硬件翻墙（路由器翻墙）

硬路由

该方式可无视各种应用层网络层硬件层的通讯协议，实现真的全局代理。路由器是本地网络接入互联网的最后一个环节，并且独立于电脑，所以可以无视流量代理规则，强制接收所有流量，来实现真全局代理。

原理可简单理解为将翻墙软件部署在了路由器上，连接路由器的设备都可以通过路由器翻墙。

优点：可应用于无法安装翻墙软件的设备上

缺点：路由器算力弱，基本只能满足最基础的路由功能，大量的加解密运算会使网络通讯产生卡顿

软路由

软路由是改装过的路由器，算力更强，克服了硬路由算力不足的短板。

缺点：价格贵；配置繁琐；稳定性一般；维护繁琐

网关模式翻墙

通过一台计算机上跑的软件来实现路由器功能，代表性产品：Windows 下的 Clash，Mac 下的 Surge。在本地局域网里部署一台运行了 Surge 或 Clash 的电脑当作网关，可接管局域网里所有网络的设备的所有流量。

优点：方便，效率高；性能良好；便于集中管理

利用ss代理配合Proxifier解决OneNote同步问题

Posted on 2019-09-03 | In 计算机网络

同步问题

OneNote，功能强大，是 Office 家族的一员，而且还免费。唯一遗憾的是，存在同步，伟大的 GFW！

个人版 OneNote 使用的 OneDrive 来同步笔记，然而受 DNS 污染等因素的影响，国内同步一直是个老大难。如果不差钱的话，可购买 OneDrive for Bussiness，商业版在国内有服务器，据说速度飞快。

准备工作

  1. 代理服务，类似 ssh,纸飞机类的代理请自备
  2. Proxifier 官方下载地址

Proxifier 为收费软件，这里提供两个注册码, 软件分为 Standard Edition 和 Portable Edition 版本，注册码不通用，注册用户名任意。

配置 Proxifier

  1. 配置代理服务器。
  • 首先打开“Proxifier”软件，然后配置文件 → 代理服务器（或者点击下图中的快捷图标）
  • 地址：127.0.0.1（这个都一样）
  • 端口：1080（你所使用的代理端口，一般是 1080）
  • 协议：SOCKS 5
  • 其他不填，然后点“确定”即可

  1. 设置规则
  • 配置文件→代理规则，进行添加规则。
  • 默认是有个名称为 Default 的规则，将其动作设为 Direct ,使电脑全部软件直接连接网络,而不通过 SS 代理联网
  • 然后点添加规则，添加一项局部代理规则，让想走代理线路的软件走代理。
    名称：任意
  • 应用程序：需要走代理的软件，这里需要代理的是 OneNote.exe
  • 动作：Proxy SOCKS5 127.0.0.1

配置好的规则列表:

  1. 设置远程 DNS

最后开启远程 DNS 解析，防止本地 DNS 污染
Profile→Name Resolution 或者点击软件左上角的 DNS，取消勾选第一个选项，选择第二个选项

  1. 运行 SS 代理

服务器软件

HTTP Server 运行于服务器之上，主要工作有：

  • 监听端口
  • 接收 socket 连接
  • 解析 HTTP 请求
  • 使用通用或专用协议对请求进行分发
  • 接收分发的请求产生的运行结果
  • 将结果格式化成 HTTP Response 并写到 socket 里面
  • 关闭连接或者 Keep-Alive

采用动静态资源分离的方式，选用 Nginx 处理静态资源请求，Tomcat 处理动态资源请求。

配置远程登录

  1. 登录 mysql

    sudo mysql -u root -p

  2. 创建新用户

    CREATE USER 'username' IDENTIFIED BY 'password';

  3. 授予远程登录权限

    GRANT ALL ON *.* TO 'username'@'%' IDENTIFIED BY 'password'

其中*.*为授权的数据库，格式为 dbname.tablename，此处为全部授权;
username 为用户名， %意味着允许用户发起远程访问的 IP 为任意 IP，也可以指定 password 为用户密码。

  1. 修改 mysql 配置文件

将/etc/mysql/my.cnf中bind_address=127.0.0.1值改为0.0.0.0。
